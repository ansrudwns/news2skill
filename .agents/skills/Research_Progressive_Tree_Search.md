# Research: Progressive Tree Search

## Trigger Condition
Activate ONLY for advanced theoretical problem-solving or Hackathons. DO NOT use for standard deterministic automation.

## Strict Research Rules
1. **Prohibition of Linear Paths:** You are strictly forbidden from pursuing a single, monolithic algorithmic approach (e.g., trying only PyTorch Gradient Descent) when facing novel mathematical operations or unknown noise parity problems.
2. **Multi-Branching:** You MUST generate at least 3 distinct theoretical approaches (e.g., Algebraic matrix mapping, Numerical brute-force, and Adaptive heuristic search).
3. **Parallel Benchmarking:** Evaluate these branches concurrently or sequentially in an isolated environment as a "Tree Search". Empirically compare their loss convergence and computational time complexity BEFORE selecting a final canonical path.
