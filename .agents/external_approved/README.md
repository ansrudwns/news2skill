# External Approved Skill Store

## Purpose
A **non-authoritative, pull-based** holding area for externally sourced skills
that have passed the `skill_triage.py` quarantine gate.

External skills here are **reference assets**, not active core instructions.
An agent must explicitly load a file from this directory to use it.

---

## Boundary with the Core SLSA Pipeline

| Property | Core skills (`.agents/skills/`) | External approved (here) |
|---|---|---|
| Source | Internally generated by LLM agent | Externally sourced `.md` files |
| Signing | `sign_drafts.py` + `.intoto.json` | Provenance frontmatter header only |
| Registry | `AGENTS.md` (authoritative) | `INDEX.json` (non-authoritative) |
| Auto-loaded | Yes, via `AGENTS.md` | No — explicit pull only |

**External skills are outside the `auto_commit.py` staging pipeline.**
`auto_commit.py` processes only `.agents/staging/draft_*.md` files and
will never touch this directory.

---

## Promotion Path

To promote an external skill into a core `.agents/skills/` asset:

```text
1. Copy the file into .agents/staging/ and rename to draft_skill_[name].md
2. python scripts/sign_drafts.py
3. python scripts/auto_commit.py    (requires user approval)
4. Update AGENTS.md manually
```

---

## Directory Structure

```text
.agents/external_approved/
  README.md         ← this file
  INDEX.json        ← discovery index (non-authoritative)
  rejected.log      ← audit rejection log
  skills/           ← imported skill files (with provenance headers)
```

---

## INDEX.json Schema

Each entry in `INDEX.json` contains only the minimum fields needed for
discovery (no full content — avoids context bloat):

```json
{
  "title": "Skill name",
  "summary": "~150 word description",
  "path": ".agents/external_approved/skills/foo.md",
  "source_path": "external_skills/foo.md",
  "risk_level": "Low | Medium",
  "warnings": ["keyword_that_triggered_P2"],
  "imported_at": "2026-04-11"
}
```

`AGENTS.md` remains the canonical core registry.
External skills are never automatically registered there.
Promotion to `AGENTS.md` requires separate explicit user approval.
